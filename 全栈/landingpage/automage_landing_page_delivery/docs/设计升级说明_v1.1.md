# 设计升级说明 v1.1

## 升级目标

在原有 `AutoMage-2 Landing Page` 工程基础上升级视觉与叙事质量，不重建项目、不改后端、不改数据库。

从「深色后台控制台风格」升级为：

- 白底企业级产品 Landing Page；
- 局部嵌入深色老板控制台预览；
- 首屏大标题 + 大留白 + 大型产品主视觉；
- 保留真实联调信息与风险披露。

## 核心变化

1. **首屏改版**
   - 大标题与产品定位左侧布局；
   - 右侧大型产品主视觉 mockup；
   - 浮动状态卡：`pytest 30 passed`、`/healthz 正常`、`主链路已跑通`、`1 个非阻塞 RBAC 风险`；
   - 增加 CTA：查看老板控制台 / 查看闭环流程。

2. **Section 节奏重构**
   - 一句话定位区（居中大字 + 三列角色分工）；
   - Workflow 视觉化；
   - Agent 卡 hover 展开关键 API/数据表；
   - 老板控制台改为深色 dashboard preview 嵌入；
   - 联调状态矩阵与数据可信管线单独成节；
   - 增加 Final CTA 收尾。

3. **新增本地 SVG 资产（无外链）**
   - `src/assets/hero-product-mockup.svg`
   - `src/assets/agent-network-visual.svg`
   - `src/assets/workflow-loop-visual.svg`
   - `src/assets/dashboard-preview-dark.svg`

4. **交互升级**
   - 导航锚点滚动；
   - Hero 浮动卡轻漂浮；
   - Workflow 步骤 hover 高亮；
   - Agent 卡 hover 展开；
   - 决策卡按钮本地状态变化；
   - Roadmap hover 动效；
   - 任务看板筛选保留。

## 保留内容（未移除）

- AutoMage-2 组织运行中枢定位；
- MVP / P0 语义；
- 主链路：Staff -> Manager -> Dream -> Decision -> Task；
- `/healthz` 正常；
- pytest 30 passed；
- 唯一非阻塞风险：`manager_cross_dept` 当前未拒绝；
- 三层 Agent、老板侧 P0、任务追踪、风险面板、数据可信链路、Roadmap。

## 运行模式与风险

- 仍保留 Demo fallback；
- API 不可用时自动回退 mock；
- 当前唯一风险继续如实展示，不做“全量完美通过”叙述。
