## 模块到依据映射

1. 顶部导航与页面定位  
   - 依据：系统总架构 + Demo 流程  
   - 理由：强调 MVP、闭环优先、老板侧展示定位。

2. Hero（主链路、健康、测试、风险）  
   - 依据：Demo 流程、Mock 全链路、联调结果口径  
   - 理由：首屏快速表达“已跑通 + 已知风险”。

3. Workflow 主链路  
   - 依据：Mock 全链路步骤（S1/M1/X1/T1/S3）  
   - 理由：每步绑定角色、表、接口、状态。

4. 三层 Agent 卡片  
   - 依据：系统总架构、三级决策机制  
   - 理由：体现分层权限和职责边界。

5. 老板侧 Dashboard  
   - 依据：Manager 汇总、Executive 决策、任务与异常 schema  
   - 理由：展示老板每天关注的核心指标与可信状态。

6. Executive 决策卡片  
   - 依据：Dream 规则 + mock_executive_decision_card_ab_options.json  
   - 理由：A/B 方案、推荐理由、确认后任务生成。

7. Manager 汇总面板  
   - 依据：mock_manager_summary_need_executive.json  
   - 理由：体现汇总与判断，而非复述日报。

8. Staff 日报与任务回流  
   - 依据：mock_staff_report_normal/high_risk.json + 任务回流机制章节  
   - 理由：体现 schema 化与回流闭环。

9. 任务看板  
   - 依据：mock_generated_tasks.json + Task schema + DDL tasks 系列表  
   - 理由：支持状态流转与筛选。

10. 风险与异常面板  
    - 依据：Staff 高风险样例 + Manager 风险汇总 + Dream 归并规则  
    - 理由：风险来源、归并、转任务链路明确。

11. API/DB/Schema 可信证明  
    - 依据：Mock 说明文档 API 映射表 + DDL + 验收标准  
    - 理由：证明页面不是静态壳。

12. Roadmap（P0/P1/P2）  
    - 依据：后续版本规划、Dream 文档扩展口、项目计划口径  
    - 理由：说明今天可交付与后续增量边界。
