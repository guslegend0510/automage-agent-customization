### block_p0208_b005

10.6任务分配规则

### block_p0208_b006

任务分配决定任务由谁执行、谁负责、谁确认。

### block_p0208_b007

MVP 阶段建议区分三类角色：

### block_p0208_b008

角色含义

### block_p0208_b009

创建方生成任务的节点或人

### block_p0208_b010

执行方实际完成任务的人或部门

### block_p0208_b011

确认方判断任务是否完成的人或节点

### block_p0208_b012

任务分配应遵守以下规则。

### block_p0208_b013

10.6.1优先分配给明确负责人

### block_p0208_b014

如果任务中已经有明确assignee_user_id，则直接分配给该用户。

### block_p0208_b015

示例：

### block_p0208_b016

{

### block_p0208_b017

"assignee_user_id": 10086,

### block_p0208_b018

"task_title": " 补充Staff Schema 错误示例"

### block_p0208_b019

}

### block_p0208_b020

这种任务可以直接推送给对应Staff Agent。

### block_p0208_b021

10.6.2只有角色时先分配给Manager

### block_p0208_b022

如果任务只指定了角色，例如backend、frontend、agent_customization，但没有明确

### block_p0208_b023

用户，则应先分配给对应部门的Manager Agent。

### block_p0208_b024

示例：

### block_p0208_b025

{

### block_p0208_b026

"assignee_role": "backend",

### block_p0208_b027

"task_title": " 补充decision_logs 表结构草案"

### block_p0208_b028

}

### block_p0208_b029

此时Manager Agent 负责将任务拆解给具体员工。
