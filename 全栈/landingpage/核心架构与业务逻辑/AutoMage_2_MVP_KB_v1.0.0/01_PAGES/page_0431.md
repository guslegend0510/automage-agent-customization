### block_p0431_b002

20.7后续屏幕记忆自动采集

### block_p0431_b003

屏幕记忆和自动采集是AutoMage-2 后续的重要增强方向，但应在三级数据闭环稳定后

### block_p0431_b004

再进入。

### block_p0431_b005

它的目标是减少员工主动填报负担，让系统能从真实工作流中自动提取部分事实。

### block_p0431_b006

可采集的数据包括：

### block_p0431_b007

数据来源可提取内容

### block_p0431_b008

文档系统文档创建、修改、评审、链接

### block_p0431_b009

代码仓库commit、PR、review、issue

### block_p0431_b010

项目管理工具任务状态、里程碑、阻塞

### block_p0431_b011

客服系统工单、投诉、处理结果

### block_p0431_b012

CRM客户跟进、商机状态、回款

### block_p0431_b013

日历会议会议主题、参与人、行动项

### block_p0431_b014

IM 群聊明确任务、风险、决策片段

### block_p0431_b015

屏幕记忆当前工作上下文、应用使用痕迹

### block_p0431_b016

但屏幕记忆必须谨慎设计。它涉及隐私、合规、员工信任和数据安全，不能简单做成“自

### block_p0431_b017

动监控”。

### block_p0431_b018

后续建议采用分阶段路线：

### block_p0431_b019

20.7.1v1.1：系统连接器

### block_p0431_b020

优先接入已有系统，而不是直接采集屏幕。

### block_p0431_b021

可先接：

### block_p0431_b022

1. GitHub / GitLab。

### block_p0431_b023

2. Jira / Linear。

### block_p0431_b024

3. 飞书文档。

### block_p0431_b025

4. Notion。

### block_p0431_b026

5. 企业微信/ 飞书群消息。

### block_p0431_b027

6. CRM 或工单系统。

### block_p0431_b028

这些系统本身就是工作事实源，采集阻力低，价值明确。

### block_p0431_b029

20.7.2v1.2：产出物自动回填

### block_p0431_b030

系统可以根据任务和日报自动关联产出物。

### block_p0431_b031

例如：

### block_p0431_b032

1. 员工完成代码提交后，Staff Agent 提醒是否关联到今日任务。

### block_p0431_b033

2. 文档修改后，自动作为任务产出物候选。

### block_p0431_b034

3. PR 合并后，自动更新任务进度。

### block_p0431_b035

4. 会议纪要生成后，自动生成待办草案。

### block_p0431_b036

这一阶段仍以“用户确认”为主，不自动替用户提交正式日报。
