### block_p0156_b003

### block_p0156_b005

### block_p0156_b006

### block_p0156_b007

8.4.4可下发部门级任务

### block_p0156_b008

Manager Agent 可以在部门权限范围内下发任务。

### block_p0156_b009

可下发任务包括：

### block_p0156_b010

1. 根据老板决策拆解出的部门内任务。

### block_p0156_b011

2. 根据Staff 日报生成的跟进任务。

### block_p0156_b012

3. 根据部门阻塞事项生成的协调任务。

### block_p0156_b013

4. 根据缺失日报生成的催办任务。

### block_p0156_b014

5. 根据次日调整建议生成的部门任务。

### block_p0156_b015

6. 根据异常处理结果生成的补救任务。

### block_p0156_b016

任务下发必须满足以下条件：

### block_p0156_b017

1. 任务属于本部门范围。

### block_p0156_b018

2. 任务不涉及超权限承诺。

### block_p0156_b019

3. 任务有明确负责人或负责人角色。

### block_p0156_b020

4. 任务有来源对象。

### block_p0156_b021

5. 任务有优先级和截止时间。

### block_p0156_b022

6. 任务状态进入数据库。

### block_p0156_b023

Manager Agent 不应只在IM 中说“大家明天处理一下”，而应生成正式Task Schema，

### block_p0156_b024

并写入任务系统。

### block_p0156_b025

8.4.5超权限事项上推老板

### block_p0156_b026

Manager Agent 必须识别超出自己权限的事项，并上推Executive Agent。

### block_p0156_b027

需要上推的典型事项包括：

### block_p0156_b028

1. 跨部门资源冲突。

### block_p0156_b029

2. 影响关键交付节点的延期。

### block_p0156_b030

3. 需要老板确认的架构或产品方向选择。

### block_p0156_b031

4. 涉及预算、采购或外部合作的事项。

### block_p0156_b032

5. 影响客户承诺的事项。

### block_p0156_b033

6. 涉及组织结构或岗位调整的事项。
