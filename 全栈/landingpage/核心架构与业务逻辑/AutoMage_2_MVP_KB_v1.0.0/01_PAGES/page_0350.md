### block_p0350_b002

16.10.2跨部门汇总

### block_p0350_b003

Executive Agent 可以读取多个部门的Manager Summary，用于生成组织级摘要。

### block_p0350_b004

Manager Agent 默认不能读取其他部门汇总，除非满足以下条件：

### block_p0350_b005

1. 该汇总被标记为跨部门协作可见。

### block_p0350_b006

2. 上级已授权。

### block_p0350_b007

3. 与当前任务或异常直接相关。

### block_p0350_b008

4. 只返回摘要级数据，不返回员工明细。

### block_p0350_b009

例如，后端组的“接口字段已确认”可以作为摘要返回给架构组，但不需要暴露后端组

### block_p0350_b010

所有员工日报。

### block_p0350_b011

16.10.3跨部门明细读取

### block_p0350_b012

跨部门员工明细默认禁止。

### block_p0350_b013

只有在极少数情况下允许：

### block_p0350_b014

1. 老板或授权Executive 下钻。

### block_p0350_b015

2. 审计人员按权限查看。

### block_p0350_b016

3. 跨部门重大异常需要核查来源。

### block_p0350_b017

4. 法务、合规或安全事件需要正式授权。

### block_p0350_b018

5. 系统管理员在排查问题时读取，且必须审计。

### block_p0350_b019

Manager 不应因为参与跨部门任务，就自动获得对方部门员工日报明细。

### block_p0350_b020

16.10.4跨部门任务权限

### block_p0350_b021

跨部门任务应拆成两个层级：

### block_p0350_b022

跨部门父任务

### block_p0350_b023

↓

### block_p0350_b024

各部门子任务

### block_p0350_b025

父任务由Executive 或项目负责人创建，分配给多个Manager。子任务由各部门Manager

### block_p0350_b026

分配给本部门Staff。

### block_p0350_b027

这样可以避免一个Manager 直接管理其他部门员工。

### block_p0350_b028

示例：

### block_p0350_b029

父任务：打通Executive 决策确认到任务生成链路

### block_p0350_b030

- 后端子任务：实现decision/commit 接口

### block_p0350_b031

- Agent 子任务：更新Executive Skill 调用逻辑

### block_p0350_b032

- 前端子任务：实现老板确认卡片按钮

### block_p0350_b033

每个Manager 只管理本部门子任务。
