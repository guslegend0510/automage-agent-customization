16.10.5跨部门授权记录

### block_p0351_b003

任何跨部门读取或协作授权，都应记录：

### block_p0351_b004

1. 授权人。

### block_p0351_b005

2. 授权对象。

### block_p0351_b006

3. 授权范围。

### block_p0351_b007

4. 授权时间。

### block_p0351_b008

5. 失效时间。

### block_p0351_b009

6. 关联任务或异常。

### block_p0351_b010

7. 读取的数据类型。

### block_p0351_b011

8. 审计日志ID。

### block_p0351_b012

MVP 阶段如果暂不做复杂授权表，也应至少在audit_logs.payload 中记录跨部门授权

原因。

### block_p0351_b014

16.11权限失败处理

### block_p0351_b015

权限失败是正常流程的一部分，不应被隐藏，也不应让Agent 反复尝试绕过。

### block_p0351_b016

权限失败主要包括401、403 和部分404。

### block_p0351_b017

16.11.1401 未认证

### block_p0351_b018

401 表示当前请求没有有效身份。

### block_p0351_b019

常见原因：

### block_p0351_b020

1. 用户未登录。

### block_p0351_b021

2. Token 过期。

### block_p0351_b022

3. IM 用户未绑定系统账号。

### block_p0351_b023

4. Agent Session 失效。

### block_p0351_b024

5. API 凭证错误。

### block_p0351_b025

处理方式：

### block_p0351_b026

1. 停止当前操作。

### block_p0351_b027

2. 暂存用户输入或Agent 草稿。

### block_p0351_b028

3. 提示用户重新登录或绑定账号。

### block_p0351_b029

4. 不写入正式数据。

### block_p0351_b030

5. 记录失败日志。

### block_p0351_b031

提示示例：

### block_p0351_b032

当前账号状态已失效，本次内容暂未提交。我已保留草稿，请重新登录后继续提交。
