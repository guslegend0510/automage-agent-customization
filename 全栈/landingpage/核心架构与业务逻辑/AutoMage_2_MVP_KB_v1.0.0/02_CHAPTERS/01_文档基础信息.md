0.7.1Agent

### block_p0030_b004

Agent 指具备一定自主执行能力的AI 节点。它可以根据预设角色、Prompt、工具权限和

### block_p0030_b005

### block_p0030_b006

在AutoMage-2 中，Agent 不是单纯聊天助手，而是组织结构中的数字节点。每个Agent

### block_p0030_b007

都有明确的角色、权限、输入、输出和可调用工具。

### block_p0030_b008

0.7.2Staff Agent

### block_p0030_b009

Staff Agent 指一线员工对应的岗位级Agent。

### block_p0030_b010

它的主要职责是辅助员工整理每日工作内容，生成符合Schema 契约的结构化日报数据，

### block_p0030_b011

并将数据提交给后端系统。Staff Agent 也可以读取上级下发给该员工的任务，并通过IM 或

### block_p0030_b012

其他交互方式提醒员工执行。

### block_p0030_b013

Staff Agent 的核心能力包括：

### block_p0030_b014

1. 引导员工填写每日工作数据。

### block_p0030_b015

2. 将自然语言内容整理为标准Staff Schema。

### block_p0030_b016

3. 提交工作记录。

### block_p0030_b017

4. 查询个人任务。

### block_p0030_b018

5. 上报阻塞问题和异常。

### block_p0030_b019

6. 在权限范围内提醒员工补充信息。

### block_p0030_b020

Staff Agent 不具备跨员工、跨部门读取数据的权限，也不应直接做重大业务决策。

### block_p0030_b021

0.7.3Manager Agent

### block_p0030_b022

Manager Agent 指部门负责人或中层管理者对应的管理级Agent。

### block_p0030_b023

它的主要职责是读取本部门Staff Agent 提交的工作数据，进行部门级汇总、风险识别、

### block_p0030_b024

异常归类、业绩标记和权限内决策，并生成Manager Schema 写入系统。

### block_p0030_b025

Manager Agent 的核心能力包括：

### block_p0030_b026

1. 读取本部门员工日报数据。

### block_p0030_b027

2. 聚合部门每日工作进展。

### block_p0030_b028

3. 识别部门风险和阻塞事项。

### block_p0030_b029

4. 标记表现突出员工和异常员工。

### block_p0030_b030

5. 处理权限范围内的常规管理决策。

### block_p0030_b031

6. 将超出权限的事项上推给Executive Agent。

### block_p0030_b032

7. 根据老板确认的决策拆解部门任务。

### block_p0032_b005

### block_p0032_b007

### block_p0032_b008

### block_p0032_b009

### block_p0032_b010

### block_p0032_b012

### block_p0032_b014

### block_p0032_b015
