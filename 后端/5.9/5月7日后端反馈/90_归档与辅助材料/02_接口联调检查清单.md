# 接口联调检查清单

用途：
- 前后端联调时逐项确认
- 避免继续出现“接口没问题，但身份口径拿错”的误判

---

## 一、联调前先确认环境

- [ ] 当前打的是 Mock/测试环境，还是当前本地真实环境
- [ ] 如果打的是当前本地真实环境，账号统一使用：
  - `zhangsan`
  - `lijingli`
  - `chenzong`
- [ ] `org_id` 统一为 `org_automage_mvp`
- [ ] `department_id / dept_id` 统一为 `dept_mvp_core`
- [ ] `node_id` 统一为：
  - `staff_agent_mvp_001`
  - `manager_agent_mvp_001`
  - `executive_agent_boss_001`
- [ ] 不再混用 `user_manager_001 / user_agent_001 / user_backend_001 / user_executive_001`

---

## 二、请求头检查

- [ ] `Authorization` 已传
- [ ] `X-User-Id` 已传
- [ ] `X-Role` 已传
- [ ] `X-Node-Id` 已传
- [ ] `X-Level` 已传
- [ ] `X-Department-Id` 已传
- [ ] `X-Manager-Node-Id` 已传
- [ ] `X-Request-Id` 已传

---

## 三、Staff 日报链路

### POST /api/v1/report/staff

- [ ] 调用身份使用 `zhangsan`
- [ ] `identity.user_id == X-User-Id`
- [ ] `identity.node_id == X-Node-Id`
- [ ] `report.org_id == org_automage_mvp`
- [ ] `report.department_id == dept_mvp_core`
- [ ] `report.record_date` 已传
- [ ] 返回 `200`
- [ ] 返回中能拿到 `work_record_id / work_record_public_id / staff_report_id`

### GET /api/v1/report/staff

- [ ] 查询参数 `org_id=org_automage_mvp`
- [ ] 查询参数 `department_id=dept_mvp_core`
- [ ] 查询参数 `record_date` 正确
- [ ] 如按人过滤，`user_id=zhangsan`
- [ ] 返回 `200`
- [ ] `data.reports[]` 可正常解析

---

- [ ] 查询参数 `summary_date` 正确
- [ ] 查询参数 `dept_id=dept_mvp_core`

---

## 五、Task 链路

### POST /api/v1/tasks

- [ ] 调用身份使用 `lijingli` 或 `chenzong`
- [ ] `task_id` 明确且唯一
- [ ] `org_id=org_automage_mvp`
- [ ] `department_id=dept_mvp_core`
- [ ] `assignee_user_id` 当前建议使用 `zhangsan`
- [ ] 返回 `200`
- [ ] 返回里能拿到 `data.tasks[]`

### PATCH /api/v1/tasks/{task_id}

- [ ] 使用业务 `task_id` 更新，不是数据库主键 id
- [ ] 调用身份与任务权限匹配
- [ ] 更新字段至少有一个
- [ ] 返回 `200`
- [ ] 返回里 `data.task.task_id` 与路径参数一致

### GET /api/v1/tasks

- [ ] Staff 查询时 `assignee_user_id=zhangsan`
- [ ] 如按状态过滤，`status=pending / in_progress / done / closed`
- [ ] 返回 `200`
- [ ] `data.tasks[]` 可正常解析
- [ ] 不再按 `task_queue` 主读源理解

---

## 六、出现问题时先排查什么

- [ ] 先确认是不是拿错了账号口径
- [ ] 再确认 `user_id / node_id / manager_node_id / department_id` 是否一致
- [ ] 再确认请求头身份和请求体身份是否一致
- [ ] 最后才判断是不是后端接口问题

---

如果打的是当前本地真实环境，就统一使用 `zhangsan / lijingli / chenzong + org_automage_mvp + dept_mvp_core + staff_agent_mvp_001 / manager_agent_mvp_001 / executive_agent_boss_001`，不要再混用旧 Mock 示例账号。
