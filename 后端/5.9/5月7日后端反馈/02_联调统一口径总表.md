# 联调统一口径总表
日期：2026-05-07

用途：
- 这份文件是当前联调唯一真源
- 目标是彻底解决“账号、`user_id`、`node_id`、部门 ID 混用”问题
- 前端、全栈、产品、文档 Owner 都应以这份表为准

---

## 一、先说结论

当前最大的联调风险不是接口本身，而是把两套身份口径混着用：

1. Mock / 测试文档口径
2. 当前本地真实库口径

如果前端打的是当前本地真实环境 `http://localhost:8000`，就必须优先使用“当前本地真实库口径”，不要直接照抄旧文档里的 `user_manager_001` 这类示例账号。

---

## 三、当前统一执行规则

### 1. 如果联调的是当前本地真实环境

默认按下面这套走：

- Staff：`zhangsan`
- Manager：`lijingli`
- Executive：`chenzong`
- `org_id=org_automage_mvp`
- `department_id=dept_mvp_core`
- `staff_agent_mvp_001 -> manager_agent_mvp_001 -> executive_agent_boss_001`

### 2. 如果联调的是 Mock 文件或隔离测试

才使用下面这套：

- `user_agent_001`
- `user_backend_001`
- `user_manager_001`
- `user_executive_001`

### 3. 两套口径不能混用

典型错误示例：

1. 请求头用 `user_manager_001`，但打的是当前真实本地库
2. 请求体里 `manager_user_id=user_manager_001`，但查询接口又改成 `manager_user_id=lijingli`
3. Staff 用真实账号，Task assignee 又用旧 Mock 账号

这些都会导致：

- 查不到数据
- 写入报“用户不存在”
- 写后读不回
- 误判为后端接口异常

## 四、当前接口主口径

### 1. Staff 日报

前端只认：

- `POST /api/v1/report/staff`
- `GET /api/v1/report/staff`

不要求前端理解：

- `staff_reports`
- `work_records`
- `work_record_items`

### 2. Manager 汇总

前端只认：

- `POST /api/v1/report/manager`
- `GET /api/v1/report/manager`

当前联调读取口径以 `manager_reports` 快照能力为准。

### 3. Task

前端只认：

- `POST /api/v1/tasks`
- `PATCH /api/v1/tasks/{task_id}`
- `GET /api/v1/tasks`

当前正式任务主域是：

- `tasks`
- `task_assignments`
- `task_updates`

`task_queue` 只保留兼容镜像语义，不再作为正式读源。

### 4. Task 更新标识

当前统一按业务 `task_id` 更新。

不要再把“数据库主键 id”和“业务 task_id”混着理解。

## 六、给前端的直接执行规则

如果前端现在要开始联调，请直接按下面 5 条执行：

1. 打 `localhost:8000` 时，一律优先用 `zhangsan / lijingli / chenzong`
2. `org_id` 一律用 `org_automage_mvp`
3. `department_id / dept_id` 一律用 `dept_mvp_core`
4. `node_id` 一律用 `staff_agent_mvp_001 / manager_agent_mvp_001 / executive_agent_boss_001`
5. 任务链路一律按 `tasks` 主域理解，不再按 `task_queue` 主域理解

## 七、给全栈的一句话结论

当前统一联调口径应明确区分“Mock 文档身份”与“真实本地库身份”：如果联调目标是当前本地真实环境，就统一使用 `zhangsan / lijingli / chenzong + org_automage_mvp + dept_mvp_core + staff_agent_mvp_001 / manager_agent_mvp_001 / executive_agent_boss_001` 这套口径，不再混用 `user_manager_001` 等旧 Mock 示例账号。
