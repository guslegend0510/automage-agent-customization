# 数据库接口交付材料对齐说明
日期：2026-05-08

## 一、文档目的

本文档用于记录 `delivery` 目录下数据库接口交付材料的对齐修正情况。

本次修正的目标是：

- 使交付材料与当前正式后端实现保持一致
- 清理旧快照表主链路表述带来的理解偏差
- 补充当前正式主链路、审计、权限控制和 Redis 版防滥用等最新能力说明

---

## 二、问题背景

检查 `automage-agent-customization-main/delivery` 目录后发现，原有数据库接口交付材料仍停留在较早版本的实现口径，主要问题包括：

1. 仍将 `staff_reports / manager_reports / task_queue` 视为主链路数据源
2. 未切换至当前正式业务表主链路
3. 未体现 `GET /api/v1/audit-logs`
4. 未体现 `POST /internal/dream/run`
5. 未体现当前权限控制、RBAC、限流和幂等能力
6. 未明确区分 HTTP 正式链路与 SQLAlchemy 本地直连链路的边界

如继续直接使用旧版材料，容易造成：

- 联调口径不一致
- 数据表理解偏差
- 正式落库位置判断错误
- 对当前后端能力的理解不准确

---

## 三、当前正式后端口径

本次已统一对齐为以下正式主链路：

### 1. Staff 日报

正式主表：

- `work_records`
- `work_record_items`

兼容镜像：

- `staff_reports`

补充分支：

- 高风险自动写 `incidents`
- 自动写 `incident_updates`

### 2. Manager 汇总

正式主表：

- `summaries`
- `summary_source_links`

兼容镜像：

- `manager_reports`

### 3. Executive 决策

正式主表：

- `decision_records`
- `decision_logs`

兼容镜像：

- `agent_decision_logs`

### 4. 正式任务

正式主表：

- `tasks`
- `task_assignments`
- `task_updates`

兼容镜像：

- `task_queue`

### 5. 审计

关键动作统一写入：

- `audit_logs`

## 四、本次修正内容

### 1. 总说明文档修正

已将数据库接口交付总说明调整为：

- 以 HTTP API 正式链路为主
- 明确快照表仅作为兼容镜像
- 明确 SQLAlchemy 直连仅用于本地调试或历史兼容

### 2. HTTP 链路说明修正

已补充当前真实主链路内容：

- Staff 正式日报读写
- Manager 正式汇总读写
- Dream 草案接口
- Executive 正式决策提交
- 正式任务创建、查询与更新
- 审计日志查询
- 权限控制、RBAC、限流、幂等保护说明

### 3. SQLAlchemy 直连说明修正

已明确：

- 当前该路径主要仍对应旧快照表
- 不作为当前正式联调验收主路径
- 不具备 HTTP 路径下的完整保护链

### 4. 接口与数据表映射修正

已将接口与表映射统一修正为当前正式后端口径，重点包括：

- `POST /api/v1/report/staff` -> `work_records / work_record_items`
- `POST /api/v1/report/manager` -> `summaries / summary_source_links`
- `POST /api/v1/decision/commit` -> `decision_records / decision_logs`
- `GET /api/v1/tasks` -> `tasks / task_assignments`
- `PATCH /api/v1/tasks/{task_id}` -> `tasks / task_updates`
- `GET /api/v1/audit-logs` -> `audit_logs`

## 七、结论

本次修正完成后，可以明确结论如下：

1. `delivery` 目录下的数据库接口交付材料已与当前项目正式后端实现保持一致。
2. 原先以快照表为主链路的表述已不再适用，当前统一以正式业务表为主。
3. 对外讲解时，应优先使用 HTTP API 主链路，不再以 SQLAlchemy 本地直连路径作为默认验收路径。
4. 当前交付材料已补充审计、RBAC、Redis 版限流与幂等等最新能力说明，可继续作为阶段交付材料使用。
