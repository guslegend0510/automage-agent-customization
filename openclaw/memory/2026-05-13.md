## 微信推送通道测试经过（01:15-01:29）

### 问题
首次推送老板没收到。排查发现 WeChat iLink API 需要接收方 48h 内与 Bot 有过互动才能推送。

### 解决
老板在微信给 Auto01 发了条消息 → 会话建立 → 推送立即成功（`outbound: text sent OK`）

### 教训
- 超过 48h 无互动的用户，推送会被微信静默拦截
- 网关重启会导致 WeChat 插件重新建立连接（需要几秒 start-account）
- `/internal/push/decision-card` API 在 **8080** 端口（前端 Docker），非 8000（后端 FastAPI）
